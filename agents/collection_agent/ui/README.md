# Collection Agent UI

Purpose: local debug UI to test text turns and inspect graph execution/state updates.

## First-time setup

From repo root (`/Users/saketm10/Projects/openclaw_agents`):

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -e .
```

Create/load env vars (minimum for current default config):

```bash
NVIDIA_API_KEY=nvapi-...
NVIDIA_BASE_URL=https://integrate.api.nvidia.com
```

Load `.env` values in shell:

```bash
set -a
source .env
set +a
```

## Run

From repo root:

```bash
python -m agents.collection_agent.ui.server
```

Then open:

- `http://127.0.0.1:8060/`

If you use the repo helper script (conda + `.env` loading):

```bash
./ui-render.sh
```

## What it shows

- Demo starter cards (`User A/B/C`) with credentials and case context.
- Agent-initiated opener flow (`/api/start-conversation`) so the first pitch is generated by the collections agent.
- Chat test box for one-turn or multi-turn manual testing.
- Execution chart per internal hop (node path + tool calls).
- Thinking trace from real-time node/tool/LLM events.
- Collapsible right panel with JSON snapshots:
  - Graph state (turn output keys)
  - Working memory state (session memory)

## APIs

- `POST /api/run-turn`
  - body: `{ "message": "...", "session_id": "...", "soft_cap": 10, "hard_cap": 50 }`
- `GET /api/session/{session_id}`
  - returns persisted conversation state + working memory state.
- `GET /api/demo-users`
  - returns three dummy users with credentials + linked case metadata.
- `POST /api/start-conversation`
  - body: `{ "user_code": "user_a|user_b|user_c", "session_id": "...", "soft_cap": 10, "hard_cap": 50 }`
